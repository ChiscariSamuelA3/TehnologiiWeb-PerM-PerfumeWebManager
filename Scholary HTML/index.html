Perfume Web Manager

Autori
Chiscari Samuel & Coman Eduard & Ghorbani Darius

Abstract

PerM (Perfume Web Manager) este o aplicatie web de management al resurselor dintr-o parfumerie care ofera utilizatorilor diverse tipuri de parfumuri. Utilizatorii au optiunea de a-si configura profilul si de a adauga preferinte in selectia de parfumuri. Cu ajutorul acestora, platforma le va sugera produse relevante profilului creat.

Introducere

Scopul acestei aplicatii web este de a oferi utilizatorilor, printr-o interfata user-friendly, o gama larga de parfumuri. Dupa crearea unui cont, utilizatorul isi poate configura profilul, adaugandu-si preferintele pentru produse: intensitate, aroma, ingrediente, anotimp, ocazie. Cu ajutorul optiunilor alese, platforma ii va sugera clientului cele mai potrivite parfumuri.
Prin intermediul platformei, utilizatorul poate sa caute produse prin cuvinte cheie, sa aplice filtre predefinite (price-range, cantitate, producator, rating, tip, gen) sau sa afle mai multe caracteristici despre produse.
Dupa ce un produs a fost ales, acesta poate fi plasat in cos, iar utilizatorul isi poate continua cautarile sau sa plaseze o comanda.

Structura

Pentru a realiza interfata web si pentru a implementa functionalitatile aplicatiei folosim:
3.1 HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afisate intr-un browser (sau navigator). Scopul HTML este mai degraba prezentarea informatiilor – paragrafe, fonturi, tabele s.a.m.d. – decat descrierea semanticii documentului.

3.2 CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atasa elementelor HTML prin intermediul unor fisiere externe sau direct in cadrul documentului html.

3.3 Javascript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalitati în paginile web, codul JavaScript din aceste pagini fiind rulat de catre browser.

3.4 SQLite

SQLite este o mică bibliotecă C care implementează un motor de baze de date SQL încapsulat. Oferă posibilitatea de a-l introduce în diverse sisteme și necesită zero-configurare. SQLite este diferit de majoritatea altor motoare de baze de date SQL prin aceea că a fost proiectat pentru a fi simplu.

3.5 Github

GitHub este un serviciu de gazduire web pentru proiecte de dezvoltare a software-ului care utilizeaza sistemul de control al versiunilor Git.

Interfata Aplicatiei

4.1 Index

Pagina contine catalogul general cu toate parfumurile disponibile.
Utilizatorul o poate vedea fara sa fie autentificat, poate aplica filtre asupra produselor prin alegerea brand-ului si intervalul de preturi. De asemenea, sidebar-ul cu filtre toate fi restrans sau extins, prin apasarea butoanelor corespunzatoare. Ulterior, pentru a putea aduga un parfum in cos sau in lista de favorite, acesta trebuie sa intre la Account unde are optiunile de a se conecta la un cont deja existent sau sa isi creeze unul nou. Odata inregistrat, in meniul Account utilizatorul va avea optiunile de a se deconecta (Logout) – fiind redirectionat catre pagina generala de produse sau de a-si vizita profilul (My profile). In acest moment, apasand pe butonul Add to cart, numarul de produse care se doresc a fi cumparate este contorizat si apare imediat dupa meniul Shopping Cart.
Pagina beneficiaza de un design responsive, navbar-ul fiind mutat pe verticala – vizibil doar prin apasarea butonului de langa search-box, produsele se vor restrange pana la unul pe rand, iar modul de afisare al filtrelor va fi unul mai restrans.

4.2 Shopping Cart si Preference

Aceaste pagini sunt destul de asemanatoare din punct de vedere al design-ului, fiind de asemenea responsive cu ajutorul proprietatilor grid-ului.
Shopping Cart este pagina in care o sa apara produsele ce doresc a fi cumparate si o sa poata fi accesata de un utilizator doar daca este inregistrat.
Preference o sa fie pagina in care, pe baza profilului utilizatorului, vor aparea produsele recomandate. Printre acestea, pot fi adaugate manual de catre utilizator si alte produse, prin apasarea butonului Add to favorite.
Pentru diferentierea acestor categorii de produse (recomandate si favorite), au fost folosite conventiile BEM, fiecare bloc avand modificatori specifici si diverse elemente.

4.3 Register

Pe pagina de register, utilizatorul va trebui sa completeze cele 4 campuri (Username, Email, Password, Confirm Password) pentru a-si crea contul. Campurile trebuie neaparat completate, altfel va aparea o eroare care reaminteste utilizatorului ca acele campuri nu trebuie omise. Mai mult, acesta poate apasa pe unul din cele 3 butoane pentru a alege sexul sau poate sa nu dezvaluie prin optiunea “Prefer not to say”. Odata ce toate campurile sunt completate si sexul este selectat, utilizatorul va apasa pe butonul de Register unde va fi redirectionat pe pagina de Login. In schimb, daca clientul are deja un cont inregistrat in baza de date, el poate merge direct pe acea pagina folosind butonul Go to Login.

4.4 Login

Pe pagina de login, utilizatorul are posibilitatea de a se loga pe contul sau care a fost creat pe pagina de Register. Cele doua campuri (Username, Password), asemanator paginii de Register, trebuie sa fie completate, altfel clientul va fi atentionat ca acele campuri nu trebuie lasate goale si butonul nu va functiona. Daca utilizatorul nu are un cont, atunci va apasa pe butonul de go to Register pentru a-si crea unul, altfel va apasa pe butonul de Login iar, daca informatiile din form sunt corecte cu cele salvate din baza de date, atunci el va fi redirectionat pe pagina de Profil.

4.5 Profile

Pe pagina de profil, utilizatorul isi poate vedea informatiile despre contul sau precum email-ul si username-ul sau. Numele de utilizator se afla in partea stanga sub poza clientului. Partea importanta pe pagina de profil este sectiunea de preferinte. Aici se pot selecta preferintele pentru parfumurile care il intereseaza pe clientul nostru, optiuniile sale momentan fiind: Gender, Season, Smell.
- Gender – ce tip de parfum este preferat: masculin sau de dame;
- Season – ce tip de anotimp s-ar purta acel parfum: primavara, vara, toamna, iarna;
- Smell – mirosul parfumului.

4.6 Product

Pe pagina produsului, utilizatorul poate afla mai multe detalii despre produsul dorit. Ii este prezentata o poza sugestiva, numele acestuia, descrierea si recenzii.
Ca functionalitati, pagina Product ofera optiunile de a selecta cantitatea dorita, de a adauga in cos sau la favorite produsul si de a adauga o recenzie.
